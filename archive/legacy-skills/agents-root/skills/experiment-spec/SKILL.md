---
name: experiment-spec
description: Use when a quant or product hypothesis must be turned into a structured experiment note with metrics, baseline, and pass/fail criteria.
---

Turn an idea into an Experiment specification.

Required sections:
- hypothesis
- why it may work
- dataset or user/sample source
- baseline
- primary metric
- secondary metrics
- success criteria
- failure criteria
- result_status placeholder

If the experiment is too broad:
- split it into smaller experiments
- state the split criteria

Do not:
- define metrics after the result
- leave the baseline implicit
